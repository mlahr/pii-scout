# Setup

This guide covers setup from a fresh clone. Basic text detection and the API
only need Python dependencies plus a SpaCy model. Corpus sampling, annotation,
and benchmark evaluation require local data that you provide or generate.

## Python Environment

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m spacy download en_core_web_lg
```

For the slower, more accurate SpaCy profile:

```bash
pip install -r requirements-models.txt
python -m spacy download en_core_web_trf
```

## Local Configuration

```bash
cp pii_config.example.yaml pii_config.yaml
```

`pii_config.yaml` is ignored by git. Use it for local-only paths, model
settings, keyword overrides, and optional name dictionaries.

Config files are loaded in this order unless `--config` is supplied:

1. `./pii_config.yaml`
2. `./config/pii_config.yaml`
3. `~/.pii_config.yaml`

## Optional Name Dictionaries

Dictionary-based name detection can use local first-name, last-name, and
stopword files:

```yaml
name_lists:
  first_names:
    source: "/path/to/first_names.txt"
  last_names:
    source: "/path/to/last_names.txt"
  stopwords:
    source: "/path/to/stopwords-en.json"
```

If you do not have these files, leave `name_lists` commented out and run with:

```bash
python pii_detect.py --models spacy-fast --detectors ner,regex --pretty
```

## Optional Local Data

The repository excludes generated and sensitive artifacts. Keep them outside
git, or under ignored paths such as `data/`, `annotation_set/`, `splits/`, and
`eval_report/`.

| Local artifact | Needed for | How to provide it |
| --- | --- | --- |
| PDF corpus | `index_corpus.py`, sampling, annotation pipeline | Put PDFs anywhere outside git or under ignored `data/` |
| Extracted paragraph text | `select_pii.py`, `sample_pii.py`, labeler | Generate with your own extractor command |
| `annotation_set/` | manual labeling | Generated by `sample_pii.py --annotation-dir` |
| `gold.jsonl` | validation, split, evaluation | Export from the labeler or create the documented JSONL format |
| `splits/` | dev/test evaluation | Generate with `python split.py --in gold.jsonl --out-dir splits/` |
| `eval_report/` | error analysis | Generated by `pii_detect.py --eval ...` |
| AI4Privacy JSONL/IDs | benchmark evaluation | Generate locally with `convert_ai4privacy.py` after reviewing dataset terms |
| first/last-name lists | dictionary detector and name gateway | Configure in local `pii_config.yaml` |

## PDF Text Extraction

`select_pii.py` and `sample_pii.py` can call an external extraction command.
Pass it with `--extract` or `--extract-cmd`.

The extractor should write paragraph text files under page directories:

```text
PAGE=0001/PAR=0001.txt
PAGE=0001/PAR=0002.txt
PAGE=0002/PAR=0001.txt
```

## Verify

cat examples/sample_input.txt | python pii_detect.py --models spacy-fast --pretty
