# pii-scout

A high-recall PII detection tool for document text.

pii-scout combines NER models with regex and post-processing rules to detect
names, locations, addresses, SSNs, phone numbers, account numbers, birthdates,
email addresses, and model-specific sensitive entity types. It can run as a CLI,
a FastAPI service, or as part of a PDF corpus annotation and evaluation workflow.

This public repository intentionally does not include PDF corpora, extracted
paragraph data, evaluation reports, generated name dictionaries, or benchmark
datasets. See [DATA.md](DATA.md) for data and licensing notes.

## Quick Start

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m spacy download en_core_web_lg
```

Run detection on text:

```bash
cat examples/sample_input.txt | python pii_detect.py --models spacy-fast --pretty
```

Run without optional local name dictionaries:

```bash
echo "John Smith, SSN 123-45-6789" \
  | python pii_detect.py --models spacy-fast --detectors ner,regex --pretty
```

Run the API:

```bash
PII_MODEL_PROFILE=spacy-fast uvicorn api.main:app --reload --port 9000
curl -X POST http://localhost:9000/detect \
  -H "Content-Type: application/json" \
  -d '{"text":"John Smith SSN 123-45-6789"}'
```

Run tests:

```bash
python -m pytest
```

## Detection Pipeline

pii-scout uses two main detection layers:

1. **NER models** for `PERSON` and `LOCATION`, using SpaCy, Piiranha, Ollama, or
   OpenRouter.
2. **Regex and rules** for `SSN`, `PHONE_NUMBER`, `BIRTHDATE`,
   `ACCOUNT_NUMBER`, `ADDRESS`, and `EMAIL`, with context boosting and
   false-positive filters.

Detected spans are normalized, merged, filtered by score, and returned as JSON.

## Local Data

Several workflows need local artifacts that are not committed to git:

| Local artifact | Needed for | How to provide it |
| --- | --- | --- |
| PDF corpus | indexing, sampling, annotation pipeline | Put PDFs outside git or under ignored `data/` |
| Extracted paragraph text | PII selection, sampling, labeler | Generate with your own `extract-text.sh` equivalent |
| `annotation_set/` | manual labeling | Generate with `sample_pii.py --annotation-dir` |
| `gold.jsonl` | validation, split, evaluation | Export from the labeler or create JSONL manually |
| `splits/` | dev/test evaluation | Generate with `split.py` |
| `eval_report/` | error analysis | Generated by `pii_detect.py --eval ...` |
| AI4Privacy JSONL/IDs | benchmark evaluation | Generate locally with `convert_ai4privacy.py` |
| first/last-name lists | dictionary detector and name gateway | Configure in local `pii_config.yaml` |

## Common Commands

```bash
# Copy local config template
cp pii_config.example.yaml pii_config.yaml

# Accurate SpaCy model
pip install -r requirements-models.txt
python -m spacy download en_core_web_trf

# Evaluate against a gold file
python pii_detect.py --eval gold.jsonl --ids splits/dev_ids.txt --json

# Build Docker image
docker build -t pii-scout .
```

## Limitations

Regex patterns are aggressive to favor recall, so false positives are expected.
The full PDF-to-evaluation workflow depends on local corpora, extraction output,
and annotation files that this public repository deliberately excludes.
